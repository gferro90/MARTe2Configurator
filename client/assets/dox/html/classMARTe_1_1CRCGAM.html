Detailed Description

GAM which implements the computation of a CRC checksum.

This GAM computes a CRC checksum on the InputSignals memory area and returns it on the output signal. Output types supported are: uint8 - uint16 - uint32.

The configuration setup must include:

  • The divisor Polynomial with (n+1) bits for the CRC-n type chosen. Common values for polynomial are: 0x7 (uint8), 0x1021 (uint16), 0x4C11DB7 (uint32).
  • The initial CRC value.
  • The Inverted option: 1 is you want the CRC checksum reflected, 0 otherwise.

The number of OutputSignals must be equal to 1.

The configuration syntax is (names and signal quantities are only given as an example): 

  +CRCGAM1 = {
      Class = CRCGAM
      Polynomial = 0x1021
      InitialValue = 0x0
      Inverted = 0
      InputSignals = {
          InputArea = {
              DataSource = DDB1
          }
      }
      OutputSignals = {
          crc = {
              DataSource = DDB1
              Type = uint16
          }
      }
  }
