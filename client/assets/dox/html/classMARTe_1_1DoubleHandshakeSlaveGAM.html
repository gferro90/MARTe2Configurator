Detailed Description

This GAM implements a double handshake protocol receiving commands from a master.

The DoubleHandshakeSlaveGAM listens on a number of command channels. Follows a description of the state machine that implements the double handshake for each command channel:

  • READY: the command signal is zero. Wait for channel to be triggered and when so, go to SENDING.
  • SENDING: wait that the clear trigger signal is zero. This means that the MARTe application handled the command. Route the command value to the ack signal and go to DONE.
  • DONE: wait that the command signal is reset by the master. When so, reset the ack signal and go to READY again.

When a command is received other GAMs (i.e the MessageGAM) could react. Once the command has been handled, the clear trigger signal is set to zero and the DoubleHandshakeSlaveGAM will send the acknowledge accordingly.
The DoubleHandshakeSlaveGAM recognizes the signals commands, acks and clear triggers by name convention:

  • Signals beginning with "CommandIn" are the input commands signals.
  • Signals beginning with "CleatTrigIn" are the input clear trigger signals.
  • Signals beginning with "AckOut" are the output acknowledge signals.
  • Signals beginning with "InternalState" are the output state machine state signals.
    Constraints:
  • [number of commands]==[number of acknowledges]
  • [number of commands]==[number of clear triggers]
  • [number of commands]==[number of internal states]
  • [commands type] == [acknowledges type]
  • [clear triggers type] == uint32
  • [internal states type] == uint32
  • The internal states signals have to be defined all together

The internal state signals provide in output the current state of the double handshake state machine for each channel:

  • 0 = READY
  • 1 = SENDING
  • 2 = DONE
  • 3 = ERROR
    The double handshake state machine can go in ERROR state in this cases:
  • From SENDING if the command signal changes.
  • From DONE if the command signal changes to something different than zero.
  • From SENDING or DONE if the specified timeout expires.

Follows an example of configuration. 

+DoubleHandshakeSlaveGAM = {
   Class = DoubleHandshakeGAM::DoubleHandshakeSlaveGAM
   InputSignals = {
       CommandIn = {
           Alias = Command
           NumberOfElements=2
           Type = uint32
           DataSource = FromMaster
       }
       ClearIn0 = {
           Type = uint32
           DataSource = DDB1
       }
       ClearIn1 = {
           Type = uint32
           DataSource = DDB1
       }
   }
   OutputSignals = {
       AckOut = {
           Trigger = 1
           Type = uint32
           NumberOfElements=2
           DataSource = ToSlave
       }
       InternalState0 = {
           Type = uint32
           DataSource = DDB1
       }
       InternalState1 = {
           Type = uint32
           DataSource = DDB1
       }
   }
}
