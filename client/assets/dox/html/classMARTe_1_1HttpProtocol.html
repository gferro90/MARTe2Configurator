bool MARTe::HttpProtocol::ReadHeader ( uint32  bufferReadSize = 1024u)

Reads the HTTP header.

Parses the HTTP header recognising the HTTP, GET, PUT, POST, HEAD commands. In case of POST command:
If the content type is multipart/form-data the data between boundaries is saved into variables in InputOptions block. This variables can be accessed also using the GetInputCommand method.
If the content type is application/x-www-form-urlencoded the url in the body is decoded and also in this case the variables are saved in the InputOptions block.

Parameters
[in]bufferReadSizethe size of the buffer to be used in the read operations.

Use the CompleteReadOperation method to get the unread data of the HTTP message after a ReadHeader call.

Returns
true if the read operation succeeds, false otherwise.

int8 MARTe::HttpProtocol::TextMode ( ) const

Retrieves the text mode status.

. If the client specifies an input command TextMode=[0(false),1(true)] the protocol will assume that the clients is expecting data in text mode (if 1) or in database mode (if 0) and the service (in this case HttpService) will call DataExportI::GetAsText if TextMode==1 and DataExportI::GetAsStructuredData if TextMode==0. This flag is initialised at -1, this means that the client does not assert anything about how it wants to receive the data and the service will proceed in its default mode.

Returns
-1 if the client has not specified a TextMode command, 1 if the client has specified TextMode=1 command, 0 if the client has specified TextMode==0 command.
