115  * Note: In GCC, CAS is mapped to the __sync_bool_compare_and_swap function,
116  * which is documented in builtins section for atomic access of GCC's
117  * documentation. This builtin is a full memory barrier, so no memory operand
118  * will be moved across the operation, either forward or backward. More info
119  * on https://gcc.gnu.org/onlinedocs/gcc-4.4.3/gcc/Atomic-Builtins.html#Atomic
120  * -Builtins
121  */

124  const T oldval,

129 /*
130  * Note: In GCC, CAS is mapped to the __sync_lock_test_and_set function,
131  * which is documented in builtins section for atomic access of GCC's
132  * documentation. This builtin is not a full barrier, but rather an acquire
133  * barrier. This means that references after the builtin cannot move to (or
134  * be speculated to) before the builtin, but previous memory stores may not
135  * be globally visible yet, and previous memory loads may not yet be satisfied.
136  * See more info on https://gcc.gnu.org/onlinedocs/gcc-4.4.3/gcc/Atomic-
137  * Builtins.html#Atomic-Builtins
138  */
139 template<typename T>
140 inline T XCHG(volatile T* const ptr,
141  const T val) {
