65  /*
66  * Implementation details:
67  * The implementation of this class relies on a fixed representation
68  * of the shared memory area which has to be only managed by this
69  * class. A private structure called Representation is used for
70  * representing that shared memory and managing its initialization and
71  * other stuff.
72  * Trick nº1: The interfaces SigblockConsumer and SigblockProducer
73  * both inherit privately from the Representation structure, so they
74  * do not need to hold a private member pointer to the shared memory
75  * area. That implies that no instances of them are going to be created,
76  * instead they are mapped to the shared memory area itself.
77  * Trick nº2: The Representation structure has a tail data member that
78  * is an array of chars with no elements (SDA::char8 member[]), which does
79  * not increase the size of the structure, but it allows to access the
80  * memory allocated beyond the end of the structure. This is needed
81  * because the number of signals is unknown at compile time and extra
82  * space is allocated for them at run time.
83  *
84  * Implementation warning nº1:
85  * It is essential to not add the following C++ features to classes
86  * mapped to the shared memory area (Representation, SigblockConsumer,
87  * and SigblockProducer):
88  * · Member pointers
89  * · Virtual methods
90  * · Virtual inheritance
91  * The reason behind that is that each instance of this class will be
92  * executing on different processes and, although accessing the same
93  * shared memory area, they will use different memory address spaces.
94  * So, using features that add pointers to the object (implicitly as a
95  * virtual table or explicit as as a data member pointers), it will
96  * make the program to crash by segmentation fault or data corruption.
97  *
98  * Implementation warning nº2:
99  * Related to the trick nº2, the existence of a special tail member
100  * in the Representation structure means that classes that inherit
101  * from Representation, like SigblockConsumer and SigblockProducer
102  * can not have any data members or the Representation's tail will
103  * overlap with them.
104  */
